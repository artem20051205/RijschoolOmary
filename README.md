# Rijschool Omary — сайт

Новый сайт для rijschool-omary.nl: светлый минимализм в стиле liquid glass + бэкенд для заявок.

## Запуск

Нужен только [Node.js](https://nodejs.org) (без `npm install` — зависимостей нет):

```
node server.js
```

- Сайт: http://localhost:5500
- Админка с заявками: http://localhost:5500/admin?key=omary-admin-2026
- Заявки сохраняются в `data/submissions.json` (из админки можно скачать JSON)

## Админ-ключ

По умолчанию ключ — `omary-admin-2026`. **Перед публикацией смени его**: задай переменную
окружения `ADMIN_KEY` или поменяй значение в начале `server.js`.

```
# PowerShell
$env:ADMIN_KEY = "мой-секретный-ключ"; node server.js
```

## Как это работает

- `index.html`, `styles.css`, `script.js` — сам сайт (можно захостить и без бэкенда:
  тогда форма откроет почтовый клиент вместо отправки на сервер).
- `translations.js` — переводы (NL/EN/FA/RU), подключается перед `script.js`.
- `server.js` — статика + `POST /api/contact`: валидация, honeypot против ботов,
  rate-limit (5 заявок / 10 мин с одного IP), сохранение в JSON.
- Порт: `PORT` (по умолчанию 5500).

## Две версии дизайна

- `/` — светлый liquid glass минимализм (`index.html`, `styles.css`, `script.js`)
- `/v2/` — тёмная editorial-версия в автомобильном стиле: чёрный + сигнально-жёлтый,
  крупная типографика, бегущая строка, горизонтальная лента тарифов (`v2/`)

Обе версии используют общий `translations.js`, общий бэкенд для формы и переключаются
ссылками в футере. Чтобы сделать v2 основной — поменяй папки местами или настрой
редирект в `server.js`.

## Языки

Переключатель языка в шапке сайта (иконка глобуса) — Nederlands / English / فارسی / Русский.
Персидский автоматически включает RTL-раскладку (зеркальная навигация, текст справа налево).
Язык запоминается в localStorage и при первом визите определяется по языку браузера.
Чтобы отредактировать или добавить перевод, правь `translations.js` — все тексты сайта
собраны там по разделам (nav, hero, pricing, contact и т.д.), в `index.html` они помечены
атрибутом `data-i18n="раздел.ключ"`.

## Деплой на Render

**Vercel не подходит для этого проекта** — сайт использует постоянный Node-сервер
(`server.js` с `http.createServer(...).listen(...)`) и хранит заявки в файле на диске.
Vercel — serverless-платформа: там нет ни постоянного процесса, ни постоянного диска,
поэтому сборка зависает (пытается дождаться завершения сервера, а он никогда не
завершается), а заявки с формы всё равно бы терялись. Render поддерживает обычные
Node-сервера "как есть", без переделки кода.

Шаги:

1. Зайди на [render.com](https://render.com) и залогинься через GitHub.
2. **New +** → **Web Service** → выбери репозиторий `RijschoolOmary`.
3. Render сам подхватит настройки из `render.yaml` в репозитории:
   - Runtime: Node
   - Build Command: пусто (зависимостей нет, собирать нечего)
   - Start Command: `node server.js`
4. В разделе **Environment** добавь переменную `ADMIN_KEY` — свой секретный ключ
   для страницы `/admin` (обязательно, дефолтный ключ из кода менять нужно всегда).
5. Нажми **Create Web Service** — через 1-2 минуты сайт будет доступен по адресу
   вида `https://rijschool-omary.onrender.com`. Свой домен можно подключить в
   настройках сервиса (**Settings → Custom Domains**).

**Важно про заявки:** на бесплатном тарифе Render диск не персистентный — при каждом
новом деплое (`git push`) файл `data/submissions.json` обнуляется. Между деплоями
(обычные визиты, перезапуски из-за неактивности) данные сохраняются нормально.
Периодически скачивай заявки через `/admin?key=...` → «Download as JSON», особенно
перед пушем новых изменений. Если заявки критичны — на Render есть платный
Persistent Disk, либо можно перейти на отправку заявок на email вместо/вместе с
файлом (могу настроить, если понадобится).
